# 🎯 事业构建器 - 卡牌工作流系统

一个基于卡牌组合概念的创新工作效率工具，通过拖拽卡片、创建组合来构建你的高效工作流程。

## ✨ 核心特性

### 1. 🎴 增强的卡牌系统
- **稀有度系统**: Common（普通）、Rare（稀有）、Epic（史诗）、Legendary（传说）
- **等级机制**: 每张卡片都有独立的等级（1-5级）
- **技能强度**: 可视化的技能值展示（0-100）
- **协同标签**: 每张卡片带有多个协同标签，用于计算组合加成

### 2. 🔗 智能组合系统
- **距离检测**: 当两张卡片靠近时（距离<200px）自动创建组合
- **协同加成**: 根据共同标签和技能强度自动计算组合效果
- **预设组合**: 内置20+个特殊组合，如"创新团队"、"效率大师"等
- **可视化连线**: 炫酷的渐变连线显示卡片组合关系

### 3. 📚 工作流模板
- **5大预设模板**:
  - 📅 高效工作日（专注力+时间管理+执行力）
  - 💻 产品开发冲刺（技术实力+敏捷迭代+团队协作+质量管理）
  - 📈 市场营销方案（市场洞察+品牌建设+客户关系+创意灵感）
  - 🌱 个人成长计划（持续学习+导师指导+复盘总结+突破创新）
  - 🎯 创业启动包（创意灵感+执行力+资金支持+市场洞察+战略规划）

- **一键应用**: 自动在画布中央以圆形排列模板卡片
- **难度标识**: Easy/Medium/Hard 难度分级
- **时间估算**: 每个模板都有预估完成时间

### 4. 🎨 精美的视觉设计
- **稀有度光效**: 传说和史诗卡片带有呼吸光效
- **流畅动画**: 使用 Framer Motion 实现丝滑的交互体验
- **渐变色彩**: 精心设计的配色方案
- **响应式布局**: 完美适配各种屏幕尺寸

## 🚀 快速开始

### 方式1: 仅前端模式（推荐新手）
```powershell
# 双击运行脚本
.\启动项目.ps1

# 选择选项 1 - 仅前端模式
# 此模式使用浏览器 localStorage 存储数据，无需 Docker
```

### 方式2: 完整模式（Docker + Supabase）
```powershell
# 确保 Docker Desktop 已启动
.\启动项目.ps1

# 选择选项 2 - 完整模式
# 此模式会启动完整的 Supabase 数据库
```

### 手动启动
```bash
# 安装依赖
npm install

# 启动开发服务器（模拟数据模式）
npm run dev

# 或启动 Docker Supabase
docker-compose up -d

# 等待30秒后初始化数据库
cat supabase/init.sql | docker exec -i supabase-db psql -U postgres -d postgres

# 启动前端（真实数据库模式）
VITE_USE_MOCK=false npm run dev
```

## 🎮 使用指南

### 基础操作
1. **拖动卡片**: 点击并拖动卡片到任意位置
2. **创建组合**: 将两张卡片拖到靠近的位置（距离<200px）
3. **查看卡片**: 点击卡片查看详细信息
4. **应用工作流**: 点击右侧"工作流"按钮，选择模板应用

### 高级技巧
- **批量组合**: 将多张相关卡片围成一圈，可快速创建多个组合
- **工作流定制**: 基于模板调整卡片位置，创建个性化工作流
- **稀有度搭配**: 传说级+史诗级卡片组合获得最高协同加成
- **标签匹配**: 选择拥有共同标签的卡片组合以获得额外加成

## 🛠️ 技术栈

- **前端框架**: React 18 + TypeScript
- **动画库**: Framer Motion
- **拖拽**: React DnD
- **数据库**: Supabase (PostgreSQL)
- **容器**: Docker & Docker Compose
- **构建工具**: Vite
- **样式**: CSS3 + CSS Variables

## 📊 数据库架构

### business_cards (卡片表)
- 基础信息：title, description, category, icon, color
- 定位：position_x, position_y, z_index
- 属性：rarity, level, skill_power
- 协同：synergy_tags (数组)

### card_combinations (组合表)
- 关联：card_id_1, card_id_2
- 命名：combination_name, description
- 效果：synergy_bonus, combo_effect
- 状态：is_active

### workflow_templates (工作流模板表)
- 基础：name, description, category
- 配置：card_ids (数组), difficulty, estimated_time
- 样式：icon, color

### user_progress (用户进度表)
- 跟踪：workflow_id, status
- 记录：completion_date, notes

## 🔧 配置说明

### 环境变量
创建 `.env` 文件：
```env
# 使用模拟数据（true/false）
VITE_USE_MOCK=true

# Supabase 配置（仅完整模式需要）
VITE_SUPABASE_URL=http://localhost:54324
VITE_SUPABASE_ANON_KEY=your-anon-key
```

### Docker 端口
- `54322`: PostgreSQL 数据库
- `54323`: Supabase Studio 管理界面
- `54324`: Supabase API Gateway (Kong)
- `54325`: Auth 服务
- `54326`: REST API
- `54327`: Storage API
