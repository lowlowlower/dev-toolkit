# 🎴 卡牌数据配置指南

## 两种数据源方案

你的项目现在支持两种方式：

### 方案1：JSON 文件（推荐用于开发/单机）

**优势：**
- ✅ 简单直观
- ✅ AI可以直接修改
- ✅ Git版本控制
- ✅ 无需Supabase
- ✅ 纯静态部署

**如何切换：**

在 `src/lib/supabase.ts` 中：

```typescript
// 使用 JSON 文件
const USE_MOCK = true  // 改成 true

// 或者设置环境变量
VITE_USE_MOCK=true
```

然后在 `src/data/cards.ts` 中修改卡牌数据即可。

---

### 方案2：Supabase 数据库（推荐用于生产）

**优势：**
- ✅ 可视化管理界面
- ✅ 实时更新
- ✅ 支持复杂查询
- ✅ 未来可扩展（用户系统、排行榜）

**如何切换：**

```typescript
// 使用数据库
const USE_MOCK = false  // 改成 false

// 或者
VITE_USE_MOCK=false
```

确保 Supabase 正在运行：
```bash
npx supabase status
```

---

## 🎯 选择建议

| 你的情况 | 推荐方案 |
|---------|---------|
| 个人项目/学习 | JSON 文件 |
| 快速原型 | JSON 文件 |
| AI辅助开发 | JSON 文件 |
| 需要可视化界面 | 数据库 |
| 多人协作 | 数据库 |
| 生产环境 | 数据库 |
| 需要用户系统 | 数据库 |
| 需要排行榜 | 数据库 |

---

## 📝 如何用AI修改JSON

### 直接告诉AI：

```
"把 src/data/cards.ts 中的 attack_execute 的伤害改成10"
```

AI会直接修改文件，你commit就行。

### 用数据库的话：

```
"在数据库中把 attack_execute 的伤害改成10"
```

AI会生成SQL，你还要：
1. 打开Studio
2. 执行SQL
3. 刷新页面

**→ 明显JSON更快！**

## 🤔 我的真实看法

### 对于你这个16张卡牌的项目：

**JSON 文件完全够用！**

数据库带来的是：
- ❌ 额外的复杂度
- ❌ 需要运行Supabase
- ❌ 部署更麻烦
- ❌ AI修改更麻烦

除非你计划：
- ✅ 加用户系统
- ✅ 做排行榜
- ✅ 多人对战
- ✅ 卡牌交易市场

**那再考虑数据库不迟！**
