## ⚡ 最快速的启动方式

### 🐳 Docker 完整模式（推荐）

**一键完成所有设置：**


```powershell

cd ..
```

## 🎮 典型使用流程

### 第一次使用

```powershell
# 1. 安装依赖
npm install

# 2. 一键启动 Docker Supabase
cd docker
.\一键启动.ps1

# 3. 启动前端（在新终端）
cd ..
npm run dev

# 4. 浏览器访问
# http://localhost:5173
```

### 每天开始工作

```powershell
# 启动 Docker Supabase
cd docker
.\start.ps1

# 启动前端
cd ..
npm run dev
```

### 每天结束工作

```powershell
# 停止 Docker（保留数据）
cd docker
.\stop.ps1
# 选择选项 1
```

### 遇到问题时

```powershell
cd docker

# 1. 查看状态
.\status.ps1

# 2. 查看日志
.\logs.ps1

# 3. 如果还有问题，完全重置
.\stop.ps1      # 选择 3
.\start.ps1
.\init-db.ps1
```

## 🆘 需要帮助？

### 问题排查步骤

1. **检查 Docker 是否运行**
   ```powershell
   docker info
   ```

2. **查看服务状态**
   ```powershell
   cd docker
   .\status.ps1
   ```

3. **查看错误日志**
   ```powershell
   cd docker
   .\logs.ps1
   ```

4. **完全重置**
   ```powershell
   cd docker
   .\stop.ps1     # 选择 3
   .\一键启动.ps1
   ```

### 常见错误

**"Docker 未运行"**
- 解决：启动 Docker Desktop

**"端口被占用"**
- 解决：
  ```powershell
  netstat -ano | findstr "54322"
  taskkill /PID <进程ID> /F
  ```

**"数据库连接失败"**
- 解决：等待数据库启动完成（约30秒）
